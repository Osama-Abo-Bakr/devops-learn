Implementation Approaches

Three ways to build the Skill Tree + Motivation System

A

SVG-Based Tech Tree (Recommended)

Render the tree as interactive SVG with nodes, edges, and animated unlock states. Use CSS transitions for glow/pulse effects. Lightweight, no new heavy dependencies. SVG works great for the vertical scrolling layout and connecting lines between nodes.

Pros

  • No new rendering library needed
  • Fast, smooth animations with CSS
  • Easy to make responsive
  • Small bundle size

Cons

  • Manual layout math for node positions
  • More code to write for connections
B

D3.js Tree Layout

Use D3's tree layout algorithm to auto-calculate node positions and connections. Already in the project (used for D3 diagram views). Leverages existing D3 infrastructure.

Pros

  • Auto-calculated positions
  • D3 already in the project
  • Built-in zoom/pan

Cons

  • Heavier for what's essentially a static tree
  • D3 DOM manipulation doesn't play great with React
  • Overkill for a fixed tree structure
C

React Flow Tree

Use the existing React Flow setup (already used for lesson diagrams). Custom node types for skill nodes, edges for connections. Familiar patterns from the diagram system.

Pros

  • React Flow already in the project
  • Interactive nodes out of the box
  • Custom node types are familiar

Cons

  • React Flow is designed for free-form editing — feels wrong for a fixed tree
  • Too much interactivity (dragging, selecting) for a read-only tree
  • Heavier than needed
